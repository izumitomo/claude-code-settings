# 計画書削除コマンド

Plan Modeで作成された計画書（`plans/*.md`）を削除する。
セッション中に複数回Plan Modeを使用した場合、全ての計画書を対象とする。

## 引数

`$ARGUMENTS` で削除対象を指定可能。

| 引数 | 説明 |
|------|------|
| （なし） | このセッションで作成された全計画書を自動特定して削除 |
| ファイルパスまたはファイル名 | 指定ファイルを直接削除 |

## 実行手順

### 1. 対象ファイルの特定

以下の優先順位で削除対象を特定する:

**優先順位1: 引数指定**

`$ARGUMENTS` が指定されている場合、それをファイルパスとして使用する。
- `plans/` プレフィックスがない場合は `plans/$ARGUMENTS` として解決
- `.md` 拡張子がない場合は付与

**優先順位2: 内容照合による自動特定**

引数なしの場合、会話中の計画タイトル・キーワードで `plans/` 配下のファイルを `grep` し、
内容が一致するファイルを特定する。セッション中に複数の計画書を作成した場合、
会話中に登場する全てのキーワードで照合し、該当する全ファイルを削除対象とする。

**優先順位3: ユーザーに確認**

照合で特定できない場合、`plans/` 配下の全ファイルを一覧表示し、
`AskUserQuestion` で削除対象をユーザーに選択してもらう。

### 2. 存在確認

各対象ファイルが存在するか確認する。存在しないファイルはスキップし、その旨を報告する。

### 3. 削除実行

存在する全対象ファイルを削除する:
```bash
rm <対象ファイルパス1> <対象ファイルパス2> ...
```

### 4. 完了報告

削除したファイル名の一覧を報告する。スキップしたファイルがあればその旨も報告する。
